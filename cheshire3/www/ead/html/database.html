Please ensure that you know what these operations do, and be certain that this is what you want to do before using them.

Operation Description
Re-build Rebuild all data stores and indexes using the files in the configured source directory. This includes extracting and indexing components, and subject clusters. You should use this option if you have added or edited files by any means other than this interface (e.g. FTP, direct editing of file contents).
Re-index Re-index the data which currently exists in the datastores. You should use this option if you have not added or edited any files, but wish to re-index the data for any other reason (e.g. index files have been corrupted).
Delete HTML Delete all cached static HTML (i.e. the HTML version of the records, NOT the search pages) in the configured cache directory. Also deletes any existing previews.
Build HTML Build static HTML copies of all files, in the configured cache directory. You should use this option if you have rebuilt the database so that the cached HTML is up to date.
